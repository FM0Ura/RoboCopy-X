# Logs Detalhados de Validação de Hash

## ?? Nova Funcionalidade: Visualização de Logs

O RoboCopy-X agora oferece um botão "Ver Logs" que permite visualizar todos os detalhes da operação de validação de hash em uma janela popup dedicada.

## ?? O Que São os Logs Detalhados?

Os logs detalhados fornecem informações completas sobre cada etapa da validação de hash:

- **Arquivos processados**: Nome e caminho de cada arquivo
- **Hashes calculados**: Hash completo de origem e destino para cada arquivo
- **Status de validação**: Se cada arquivo passou ou falhou na validação
- **Timestamps**: Quando cada operação foi realizada
- **Mensagens de erro**: Detalhes sobre qualquer problema encontrado

## ?? Como Usar

### 1. Executar Validação de Hash

Primeiro, execute uma operação com validação de hash ativada:

1. Ative a opção "Validar integridade com hash"
2. Selecione o algoritmo desejado (SHA-256, BLAKE3, etc.)
3. Execute a operação de cópia

### 2. Visualizar Logs Durante ou Após a Operação

Durante ou após a validação, você verá o card de "Validação de Hash" com um botão "Ver Logs":

## ?? Informações Incluídas nos Logs

### Para Cada Arquivo

| Informação | Descrição | Exemplo |
|------------|-----------|---------|
| **Nome do Arquivo** | Nome completo do arquivo | `documento.pdf` |
| **Caminho Origem** | Caminho completo do arquivo de origem | `C:\Source\documento.pdf` |
| **Caminho Destino** | Caminho completo do arquivo de destino | `D:\Backup\documento.pdf` |
| **Hash Origem** | Hash calculado do arquivo de origem | `a7ffc6f8bf1ed76...` |
| **Hash Destino** | Hash calculado do arquivo de destino | `a7ffc6f8bf1ed76...` |
| **Status** | Resultado da validação | `? Válido` ou `? Inválido` |

### Resumo Final

O resumo final inclui:
- **Total de arquivos** processados
- **Arquivos válidos** (hashes idênticos)
- **Arquivos inválidos** (hashes diferentes)
- **Arquivos ausentes** (não encontrados no destino)
- **Erros** (problemas durante o processamento)

## ?? Indicadores Visuais nos Logs

### Ícones e Símbolos

- **??** - Início da validação
- **??** - Arquivo individual
- **??** - Diretório
- **?** - Processamento em andamento
- **?** - Sucesso / Validação bem-sucedida
- **?** - Falha / Arquivo inválido
- **?** - Erro crítico
- **?** - Aviso / Arquivo ausente
- **?** - Operação cancelada

### Formatação

- **Cabeçalhos**: Linha de separação `???...???`
- **Informações principais**: Negrito ou destaque
- **Hashes**: Fonte monoespaçada (Consolas) para fácil leitura
- **Indentação**: Hierarquia visual das informações

## ?? Casos de Uso

### 1. Auditoria de Integridade

Após um backup crítico, use "Ver Logs" para:
- Verificar que todos os arquivos foram validados
- Copiar hashes para documentação
- Confirmar integridade de arquivos específicos

### 2. Troubleshooting

Se houver falhas na validação:
- Identifique exatamente quais arquivos falharam
- Compare os hashes de origem e destino
- Determine se é problema de rede, disco ou corrupção

### 3. Conformidade e Documentação

Para requisitos de conformidade:
- Exporte os logs para documentação
- Mantenha registro de validações realizadas
- Prove integridade dos dados transferidos

### 4. Verificação Rápida

Para operações rápidas:
- Verifique visualmente se a validação foi bem-sucedida
- Confirme o algoritmo usado
- Revise quaisquer avisos ou erros

## ?? Fluxo de Trabalho Recomendado

1. Execute a cópia com validação ativada

3. Clique em "Ver Logs" para confirmar sucesso

5. Investigue a causa raiz (rede, disco, etc.)
6. Re-execute a operação para arquivos específicos

## ?? Segurança e Privacidade

### O Que É Mostrado nos Logs

? **Incluído**:
- Nomes de arquivos
- Caminhos completos
- Hashes calculados
- Status de validação
- Algoritmo usado

? **NÃO Incluído**:
- Conteúdo dos arquivos
- Dados sensíveis dos arquivos
- Informações de rede
- Credenciais ou senhas

### Recomendações

- Logs podem conter caminhos completos de arquivos
- Hashes não revelam conteúdo, mas confirmam identidade
- Não compartilhe logs se os nomes/caminhos forem sensíveis
- Os hashes são seguros para documentação pública

## ?? Glossário

| Termo | Significado |
|-------|-------------|
| **Hash** | Impressão digital única do arquivo |
| **Origem** | Arquivo ou diretório sendo copiado |
| **Destino** | Localização para onde o arquivo é copiado |
| **Validação** | Processo de comparar hashes de origem e destino |
| **Íntegro** | Arquivo não foi alterado ou corrompido |
| **Corrompido** | Arquivo foi alterado durante a transferência |
